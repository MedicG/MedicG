Medic G en Vercel

Sube a GitHub el contenido de esta carpeta, no una carpeta externa que contenga otra carpeta medicg-db.

Archivos clave que deben quedar en la raiz del repositorio:
- index.html
- config.js
- vercel.json
- requirements.txt
- medicg_app.py
- api/index.py
- api/__init__.py

No debe existir en la raiz:
- server.py
- runtime.txt
- pyproject.toml
- medicg.sqlite3

Si Vercel sigue mostrando error de pyproject.toml:
- Asegurate de subir este .vercelignore actualizado.
- Si GitHub conserva un pyproject.toml viejo en la raiz, Vercel lo ignorara antes de compilar.
- Si aun falla, entra al repositorio en GitHub y borra pyproject.toml manualmente.

Variables necesarias en Vercel:
- DATABASE_URL o POSTGRES_URL: conexion PostgreSQL/Neon.
- ADMIN_EMAIL: correo inicial del administrador.
- ADMIN_PASSWORD: clave inicial del administrador.
- PUBLIC_API_URL: URL de Vercel, por ejemplo https://tu-proyecto.vercel.app
- PUBLIC_APP_URL: URL de GitHub Pages, por ejemplo https://medicg.github.io/MedicG
- ALLOWED_ORIGINS: https://medicg.github.io

Correo de verificacion:
- SMTP_HOST
- SMTP_PORT=587
- SMTP_USER
- SMTP_PASSWORD
- SMTP_FROM
- SMTP_TLS=true
- SMTP_SSL=false

Despues del deploy prueba:
https://TU-PROYECTO.vercel.app/api/health

Si responde {"ok": true, "db": "postgres"}, ya esta conectado.
Si dice "Base de datos no configurada en Vercel", falta conectar Neon/Postgres.

En GitHub Pages, config.js apunta a:
https://medic-g-git-main-medicgs-projects.vercel.app

Si tu Vercel usa otro dominio, cambia la constante MEDICG_VERCEL_BACKEND en config.js.
