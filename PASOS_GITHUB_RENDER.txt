GitHub Pages + base de datos segura

GitHub Pages solo publica archivos estaticos: HTML, CSS, JS e imagenes.
No ejecuta Python, no abre SQLite y no responde rutas /api/login.

Para guardar todo en una base de datos segura necesitas:
1. Frontend en GitHub Pages: index.html + config.js.
2. Backend en Render, Railway o VPS: server.py + base SQLite persistente.

Pasos rapidos con Render
1. Entra a https://render.com y crea una cuenta.
2. Crea un nuevo Web Service.
3. Conecta el repositorio donde subiste estos archivos.
4. Usa:
   Build Command: vacio
   Start Command: python server.py
5. En Environment Variables agrega:
   HOST=0.0.0.0
   ADMIN_EMAIL=admin@medicg.com
   ADMIN_PASSWORD=CAMBIA_ESTA_CLAVE
   MEDICG_DB_PATH=/tmp/medicg.sqlite3
   ALLOWED_ORIGINS=https://TU-USUARIO.github.io
   PUBLIC_APP_URL=https://TU-USUARIO.github.io
   SMTP_HOST=smtp.tu-proveedor.com
   SMTP_PORT=587
   SMTP_USER=tu-correo@dominio.com
   SMTP_PASSWORD=TU_CLAVE_SMTP
   SMTP_FROM=tu-correo@dominio.com
   SMTP_TLS=true
   SMTP_SSL=false
6. Render te dara una URL parecida a:
   https://medicg-backend.onrender.com
7. Edita config.js en GitHub y coloca:
   window.MEDICG_API_BASE = "https://medicg-backend.onrender.com";
   window.MEDICG_REQUIRE_BACKEND = true;
8. Espera 1 o 2 minutos a que GitHub Pages actualice.
9. Abre la pagina e inicia sesion.

Importante
- En Render gratis, /tmp puede borrarse cuando el servicio reinicia.
- Para un cliente real conviene usar un VPS o almacenamiento persistente.
- Cambia ADMIN_PASSWORD antes de entregar el sistema.
- El registro con correo necesita SMTP configurado para enviar el enlace de verificacion.
- Los usuarios registrados quedan pendientes hasta verificar correo y recibir permisos del admin.
